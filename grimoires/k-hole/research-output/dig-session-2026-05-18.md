### Findings

Siddharth Ahuja's `blender-mcp` repository established the baseline for Agent-to-Blender communication, but its reliance on raw TCP sockets routinely triggers an "Incomplete JSON response" error. Because TCP is stream-oriented, massive JSON payloads of vertex data get truncated. Engineers bypass this by implementing "Length-Prefixed Framing" — calculating the exact byte length of the payload and sending it as a fixed-size header so the receiver enters a `while` loop, continuously appending chunks until the count is reached. This mirrors the dual-layer architecture of UnrealMCP, which successfully routes massive scene graphs through a dedicated TCP bridge to circumvent standard `stdio` limits and "noise" (bridge). In high-throughput environments like algorithmic trading, length-prefixed framing (specifically the FIX protocol) has long been the gold standard for zero-loss packet reconstruction over raw TCP streams, a principle now essential for stable 3D agent payloads (adjacent).

Blender's Python API (`bpy`) strictly binds to the main thread, meaning standard blocking socket listeners will instantly freeze the host UI. To maintain Python addon health, practitioners spin up the socket or local REST server in a background daemon thread that pushes agent commands into a `queue.Queue`. Back on the main thread, the addon uses `bpy.app.timers.register` to run a polling function every 0.1 seconds, executing the commands safely. Robust setups further isolate this execution by running the agent's `bpy.ops` inside a `try/except` block, capturing the exact traceback and returning it so the agent can self-correct its syntax without crashing the connection. 

Kunal Gupta's **InteriorAgent** framework points away from letting LLMs blindly guess arbitrary syntax, utilizing Domain Specific Languages (DSLs) to map high-level reasoning to low-level spatial rules. This philosophy is emerging in production-grade implementations like `blender-ai-mcp` and `RevitMCP`, which expose a "strict, curated API" rather than raw code execution. These setups replace massive geometric arrays with simple file-system pointers (like USD/FBX paths) and goal-first routing. The Blender Artists community's `Blender MCP Pro` pushes this further, employing lazy loading across over 100 tools to preserve the AI context window. Using intermediate DSLs and strict APIs to constrain LLM hallucinations mirrors how code-generation agents in web development target abstract syntax trees (ASTs) rather than emitting raw string concatenations, ensuring structural validity before execution (adjacent).

### Emergence

The evolution of 3D MCP servers is moving in the exact opposite direction of general-purpose coding agents. While tools like Cursor prioritize giving models unrestricted, raw read/write access to everything, agentic 3D architectures only succeed through severe limitation: bottlenecking network communication into background queues, replacing raw coordinate arrays with file pointers, and stripping away arbitrary code execution in favor of strict, curated APIs. Constraint, rather than open access, is the prerequisite for stability in spatial environments.
